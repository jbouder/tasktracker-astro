These are the **actual** [Nebari design-system](https://github.com/nebari-dev/nebari-design)
React components the Tasker app ships — installed from the `@nebari` shadcn
registry and rendered live on this page. The docs and the app share one set of
brand tokens, so what you see here matches the product exactly. Toggle the
theme switch in the header and watch every component re-theme.

<Aside type="tip">
Each demo below is a hydrated React island (`client:load`), so the buttons,
toggles, and links are genuinely interactive — not screenshots.
</Aside>

## Buttons

The `Button` ships six variants, four sizes, a `loading` state, and Base UI
`render`-prop composition (render it as a link without losing its styling).

## How this works

<CardGrid>
	<Card title="Shared tokens" icon="puzzle">
		The Nebari OKLCH color tokens live in `src/styles/global.css` and are also
		mapped onto Starlight's `--sl-*` chrome variables — one palette, whole site.
	</Card>
	<Card title="Installed, not forked" icon="approve-check">
		Components come from the `@nebari` registry and are treated as
		upstream-managed. We style at the call site via `className` and the `render`
		prop — never by editing `src/components/ui/*`.
	</Card>
</CardGrid>
